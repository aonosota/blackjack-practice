# 🃏 ブラックジャック ストラテジー練習

ベーシックストラテジーを身につけるためのブラウザ練習アプリです。
サーバー不要。`index.html` をブラウザで開くだけで動作します。

## デモ

**GitHub Pages で公開中 → [https://aonosota.github.io/blackjack-practice/](https://aonosota.github.io/blackjack-practice/)**

ブラウザからそのまま利用できます。インストール不要。

## 特徴

- ベーシックストラテジー通りのアクションを判定・フィードバック
- ハンド終了後に**多角的な分析**と**ストラテジー表**を表示
- ルール設定（デッキ数・S17/H17・DAS・サレンダー）に応じて推奨アクションが変化
- プレイ記録を**CSV（BOM付きUTF-8）**でダウンロード可能

## 使い方

1. `index.html` をブラウザで開く
2. 必要に応じて設定を変更し「設定を適用」
3. 「セッション開始」でスタート
4. ベットを設定して「ディール」
5. ヒット・スタンド・ダブル・スプリット・サレンダーを選択
6. ハンド終了後に分析とストラテジー表を確認
7. 「セッション終了 & CSV保存」でプレイ記録をダウンロード

## ルール設定

| 設定 | 選択肢 | デフォルト |
|------|--------|-----------|
| デッキ数 | 1 / 2 / 4 / **6** / 8 | 6デッキ |
| ディーラーのソフト17 | **S17**（スタンド） / H17（ヒット） | S17 |
| DAS（スプリット後ダブル） | **あり** / なし | あり |
| サレンダー | **なし** / あり（レイトサレンダー） | なし |
| 開始チップ | 任意（最小100） | ¥10,000 |
| 最小/最大ベット | 任意 | ¥500 / ¥10,000 |

## ハンド分析

ハンド終了後に以下の情報が表示されます。

- ✅ / ❌ / △ の正誤判定
- 推奨アクションとその理由
- 境界ケース（EV差が僅かな局面）の注記
- ルール変更（S17/H17・DAS・サレンダー）による推奨の違い
- よくある間違いの指摘
- 現在のルール設定を反映した**ストラテジー表**（該当セルをハイライト）

## CSV 記録フォーマット

| 列 | 内容 |
|----|------|
| 日時 | アクション発生時刻 |
| ハンド番号 / アクション番号 | セッション内の連番 |
| ディーラーオープンカード | ディーラーのアップカード |
| プレイヤーカード / 合計 / ハンド種別 | プレイヤーの手札情報 |
| 取ったアクション / 推奨(表記) / 推奨(実行可能) / 評価 | アクション判定 |
| デッキ数 / ディーラールール / DAS / サレンダー | ルール設定 |
| ベット額 / ハンド結果 / 収支 | 金額情報 |

## ファイル構成

```
blackjack_practice/
├── index.html        # メイン HTML
├── css/
│   └── style.css     # カジノ風 UI スタイル
└── js/
    ├── strategy.js   # ストラテジー表・推奨アクション取得
    ├── game.js       # ゲームロジック（カード・デッキ・精算）
    ├── analysis.js   # ハンド後分析テキスト生成
    ├── csv.js        # CSV 記録・ダウンロード
    └── main.js       # UI コントローラー・ゲームフロー
```

## アクションコード（ストラテジー表）

| コード | 意味 |
|--------|------|
| H | ヒット |
| S | スタンド |
| D | ダブルダウン（できなければヒット） |
| Ds | ダブルダウン（できなければスタンド） |
| P | スプリット |
| R | サレンダー（できなければヒット） |
| Rs | サレンダー（できなければスタンド） |
| Rp | サレンダー（できなければスプリット） |
